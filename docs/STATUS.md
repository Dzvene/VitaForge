# Build status — honest assessment

Sober assessment against the full v1 scope, not hype. As of the initial scaffold.

## Done and verified

- **Backend architecture**: VSA, 9 slices, `tach check` green (boundaries
  enforced, graph acyclic). App imports cleanly, 36 routes wired.
- **Pure calculation engine** (`app/core/nutrition_math.py`) — the product's
  value — fully implemented and unit-tested (19 tests pass):
  - §4.1 Mifflin-St Jeor BMR + activity TDEE
  - §4.2 macros (protein priority, fat floor, carb fill, abs/per-kg overrides)
  - §4.3 EMA weight trend
  - §4.4 real-TDEE back-calculation
  - §4.4/§9 goal target with the ≤1%/week safety clamp + gain surplus band
- **Configurable params** (`app/core/params.py`) — every §6 threshold, with
  per-user override merge. Nothing hardcoded in the engine.
- **All slices have working CRUD/logic + endpoints**: auth, profile, nutrition
  (Norm), foods (search/barcode/custom/favorites), diary (log/summary/copy/
  recent), weight (log/series+trend), calibration (status/estimate/recalc/skip),
  coaching (hints/warnings/day-guidance/accept/dismiss).
- **Onboarding wiring** via in-process events (profile change → recompute norm +
  open calibration window), no cross-slice cycle.
- **Initial Alembic migration** autogenerated (all 10 tables).

## Real but thin / needs work

- **No API-level integration tests yet** — only the engine is tested. The slice
  services are exercised by import + types, not by end-to-end request tests.
- **Coaching rules are a first pass** — triggers and in-day guidance thresholds
  are reasonable but unvalidated against real usage; copy is placeholder-quality.
- **Food import is a scaffold** — OFF mapper only, USDA not implemented, no
  dump downloaded, no full-text search index (uses ILIKE).
- **Calibration estimate** uses a simple trailing-span model; the "soft degrade"
  on dirty data is implemented but the exact gap thresholds want field tuning.

## Not started

- **Mobile apps** — iOS (Swift/SwiftUI) first, then Android (Kotlin/Compose),
  per the Invocore order. Folders not yet scaffolded. Backend has no OpenAPI
  hand-off doc for the native DTOs yet.
- v2 (auto weekly adaptive TDEE, analytics) and v3 (training) — out of scope now.

## Honest one-liner

The brain (calculation engine + clean architecture) is real, tested, and
correct. The body (food data at scale, integration tests, polished coaching copy)
and the face (mobile apps) are not built yet.
